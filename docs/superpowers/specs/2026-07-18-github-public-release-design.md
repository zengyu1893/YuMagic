# GitHub 公开发布设计

**日期：** 2026-07-18
**状态：** 方案 A 已批准

## 目标

将当前 Penguin Magic 源代码发布到 `zengyu1893/Penguin-Magic` 公开仓库，确保不暴露凭据、本地用户数据、生成内容或现有本地 Git 历史。

## 风险范围

公开发布内容不得包含：

- API Key、访问令牌、密码、授权请求头、私钥或发布凭据。
- 可能包含供应商配置或用户内容的本地设置和运行时数据。
- 临时上传文件、生成记录、生成输出、进程 ID、Agent worktree 和本地工具元数据。
- 包含凭据格式占位符或历史本地文件的旧提交。
- 提交者的个人 Git 邮箱地址。

公开的供应商基础地址和注册地址属于文档信息，不属于秘密。

## 选定方案

从清理后的当前工作区生成一个全新的单提交公开快照。现有本地分支和提交图保持不变，不上传旧历史；通过备用索引生成独立 Git 引用，再推送为 GitHub 的 `main` 分支。

相比重写现有历史，这种方式不会扰动当前多个 worktree 和大量未提交开发修改。相比直接推送，它能避免只删除最新文件却仍在历史提交中泄露敏感信息。

## 仓库内容调整

1. 将 API Key 格式的界面占位文本替换为中性的示例文本。
2. 扩展 `.gitignore`，忽略后端本地数据、临时上传、生成记录、进程 ID、Agent worktree、本地工具元数据、凭据和构建产物。
3. 需要时保留项目内运行目录的 `.gitkeep`，不把运行数据迁移到项目外。
4. 在 README 增加以下两个 API 基础地址及 Key 注册入口：
   - `https://yuli.host`，注册：`https://yuli.host/register?aff=64350e39653`
   - `https://ai.yuliapi.com`，注册：`https://ai.yuliapi.com/register`
5. 新增 `SECURITY.md`，说明凭据只保存在本地以及私密漏洞反馈方式。
6. 更新 `docs/file-index.md` 和 `.claude/rules/ecc/penguin/file-map.md` 的持久文件登记。

被忽略的本地文件继续留在项目目录内，只从公开快照中排除。

## GitHub 发布

在已登录的 GitHub 账号 `zengyu1893` 下创建公开仓库 `Penguin-Magic`，创建时不自动生成 README、`.gitignore` 或许可证。使用通用项目身份和非个人邮箱创建公开根提交，只将该根提交推送到 `refs/heads/main`。

推送成功后，将本地 `origin` 指向新仓库，但保留当前本地开发分支和历史不变。

## 验证

发布前：

- 运行项目构建和现有测试。
- 对准确的公开候选树扫描常见凭据模式。
- 确认被排除的运行时路径不在候选树中。
- 确认候选提交只有一个提交，且使用非个人提交身份。

发布后：

- 确认 GitHub 仓库为公开状态，默认分支为 `main`。
- 确认 README 中两个 API 地址可以正常显示。
- 再次扫描已推送的 `main`，确认本地敏感路径不存在。

## 失败处理与回滚

扫描或验证失败时不推送，修正候选快照后重新生成。仓库创建成功但推送失败时保留空仓库，待认证恢复后重试。若推送后发现敏感内容，立即撤销公开访问或删除新仓库，撤销已暴露凭据，再次完整扫描后只发布新的干净根提交。
